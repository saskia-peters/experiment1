## Executive Summary

The application is well-structured for a single-developer Wails v2 desktop project of this scope. The five-layer backend, consistent IPC response contract, transactional database inserts, and modular ES6 frontend are all positive. However, **three XSS/injection vulnerabilities** in the admin UI, **one path traversal** in the restore handler, **one panic risk** from missing config validation, and **one archived critical dependency** require attention before the next release.

**Critical / High issues: 11**  
**Medium issues: 19**  
**Low / Polish issues: 22**

## Architecture Review

### Strengths

**A-S1 Clean Five-Layer Separation**  
The backend follows a strict layer model: Wails bindings (`app_handlers.go`) → Application handlers (`backend/handlers/`) → Database (`backend/database/`) → Domain service (`backend/services/`) → I/O (`backend/io/`). `app_handlers.go` is a pure delegation adapter with zero logic — this is correct Wails usage and all business logic is testable without a Wails context.

**A-S2 Consistent IPC Response Contract**  
Every Wails-bound method returns `map[string]interface{}` with a `"status"` key (`"ok"` or `"error"`). ~20 endpoints follow this uniformly; frontend error handling is consistent throughout.

**A-S3 Input Validation at System Boundaries**  
`backend/io/input.go` validates Excel input (header names, age range 0–150, gender allow-list, alphanumeric PreGroup) before any data reaches the database. Validation occurs at the correct layer.

**A-S4 Transactional Bulk Inserts + Prepared Statements**  
All `backend/database/inserts.go` functions use explicit transactions with `defer tx.Rollback()`, prepared statements, and `?`-bound parameters. SQL injection is not possible through this path.

**A-S5 Foreign Key Enforcement**  
Both `OpenExistingDB` and `InitDatabase` immediately execute `PRAGMA foreign_keys = ON`. This is the correct position — SQLite resets this pragma per connection.

**A-S6 PDF Theme Centralization**  
`backend/io/pdf_styles.go` defines `PDFTheme` as a single design token system. One change propagates to all five PDF generators.

**A-S7 TOML-Driven Certificate Layout**  
`backend/io/cert_layout.go` externalizes layout to a user-editable file with auto-generated defaults and a graphical editor in the frontend. The sentinel values (`-1` for auto-x, `0` for auto-width) are well-documented.

**A-S8 Pre-Restore Backup Integrity Check**  
`backend/handlers/backup.go` runs `PRAGMA quick_check` on the backup file before removing the live database. A corrupt backup is rejected before the destructive operation.

**A-S9 Dirty-State Tracking in Score Entry**  
`frontend/stations/stations.js` tracks unsaved changes via `savedScoreMap` and confirms navigation away from a dirty group. No silent data loss.

### Architecture Issues

#### A-1 — CRITICAL: Missing Config Semantic Validation → Panic Risk
**Files:** `backend/config/config.go`, `backend/services/distribution.go`

`ValidateAndSave` checks TOML syntax only. `max_groesse = 0` is accepted and silently stored. At the next "Gruppen zusammenstellen" click:

```go
// distribution.go
numAdditionalGroups := int(math.Ceil(float64(len(unassigned)) / float64(maxGroupSize)))
// maxGroupSize=0 → float64(0) → +Inf → int(+Inf) = math.MinInt64 on amd64
// make([]models.Group, math.MinInt64) → panic: runtime error: makeslice: len out of range
```

Similarly, `min_punkte >= max_punkte` causes every score submission to be rejected with no explanation.

**Recommendation:** Add `func (c Config) Validate() error` checking `max_groesse >= 1` and `max_punkte > min_punkte`. Call it from `ValidateAndSave` and `LoadOrCreate`. Add a guard at the top of `distributeIntoGroups`.

---

#### A-2 — HIGH: Path Traversal in `RestoreDatabase`
**File:** `backend/handlers/backup.go`, ~line 68

```go
backupPath := filepath.Join("dbbackups", backupFilename)
// filepath.Join("dbbackups", "../config.toml") → "config.toml"
// If this happens to be a valid SQLite DB, quick_check passes, live DB is deleted
// and overwritten with the traversed file.
```

The same codebase correctly applies the canonical path containment check in `GetGroupPictureAsBase64` (`filepath.Abs` + `strings.HasPrefix`). Apply the same guard here before any `os.Stat` call.

---

#### A-3 — HIGH: Startup Errors Are Silent in Production (Desktop App Has No Console)
**Files:** `main.go`, `backend/services/distribution.go`

Config load failure and directory creation failure are reported via `fmt.Printf`. A Wails desktop app on Windows has no console window; stdout is discarded. An operator with a hand-edited, syntactically broken `config.toml` gets no indication it was ignored.

`backend/services/distribution.go` also contains `fmt.Printf` for production diagnostic output (group creation results).

**Recommendation:** Add `startupWarnings []string` to the `App` struct. Return warnings from `GetConfig()` so the frontend can display them. Replace all `fmt.Printf` in service/business code with `log/slog`.

---

#### A-4 — MEDIUM: `models.DbFile` Is a Mutable Exported Global
**File:** `backend/models/types.go`

```go
var DbFile = "data.db"  // written in main, read by 6 packages
```

This global is not updated when `SaveConfigRaw` changes `a.cfg.Ausgabe.DBName` at runtime, creating inconsistency. It also creates an implicit coupling between unrelated packages. Pass the DB path as a parameter to `OpenExistingDB` and `InitDatabase`; the `App` struct already has `a.cfg.Ausgabe.DBName`.

---

#### A-5 — MEDIUM: UI Button State Managed at Six Independent Call Sites
**Files:** `frontend/app.js`, `frontend/admin/file-handler.js`, `frontend/stations/stations.js`

Button enable/disable logic is duplicated across six locations. The logic for which buttons are active after "scores exist" differs subtly between paths (e.g., `btnPDF` state differs between file-load and restore paths). New lifecycle events will continue to diverge.

**Recommendation:** Introduce a single `applyState(state)` function in `shared/state.js` that takes the current application lifecycle state (`dbLoaded`, `groupsExist`, `scoresExist`) and derives all button states from it. Every lifecycle event mutates state and calls `applyState()`.

---

#### A-6 — MEDIUM: SQLite Backup Is a Raw File Copy Without WAL Consideration
**File:** `backend/handlers/backup.go`

The backup uses `io.Copy` on the `.db` file while the live `*sql.DB` connection remains open. This is safe with the default DELETE journal mode. If `journal_mode` is ever changed to WAL, the `.wal` sidecar file would not be included, producing an inconsistent backup. The live connection also has a narrow race window with concurrent writes during copy.

**Recommendation:** Issue `PRAGMA wal_checkpoint(FULL)` before copying. At minimum add a comment documenting the WAL assumption.

---

#### A-7 — MEDIUM: `InitDatabase` DDL Execution Is Not Atomic
**File:** `backend/database/db.go`

11+ `CREATE TABLE` and `DELETE FROM` statements are executed as individual `db.Exec` calls with no enclosing transaction. A process kill mid-way leaves the database in an inconsistent partially-wiped state. SQLite supports transactional DDL; wrap the entire body in `BEGIN IMMEDIATE … COMMIT`.

---

#### A-8 — MEDIUM: Config Load Failure Leaves Corrupt File on Disk
**File:** `backend/config/config.go`

When `toml.DecodeFile` fails, the function returns defaults and logs (silently in production — see A-3). The invalid `config.toml` is left on disk and will fail again on every subsequent launch.

**Recommendation:** On decode failure, rename `config.toml` to `config.toml.bak.YYYYMMDD`, write fresh defaults, and surface the warning to the user.

---

#### A-9 — MEDIUM: Misleading Config Save Success Message
**File:** `backend/handlers/admin.go`, line ~51

The message "Einige Änderungen (z. B. Gruppen, Ergebnisse) werden erst nach einem Neustart der App wirksam" is incorrect. `a.cfg` is updated in `app_handlers.go` immediately on save — `MinPunkte`/`MaxPunkte` apply to the next `AssignScore` without restart. Only `DBName` requires restart. The message causes unnecessary restarts by operators.

#### A-10 — LOW: Schema Design — No FK Constraint on `group_station_scores.group_id`
**File:** `backend/database/db.go`

The code comment acknowledges the missing FK. Scores can be inserted for group IDs with no participants, with no database-level error. The correct fix is a separate `groups(group_id INTEGER PRIMARY KEY)` table that both `gruppe` and `group_station_scores` reference.

## Go Backend Review

### Go Idiom Issues

#### G-1 — HIGH: `InitDatabase` Leaks `*sql.DB` on Error Paths
**File:** `backend/database/db.go`

All early-return error paths after `sql.Open` fail to call `db.Close()`. The correct pattern:

```go
func InitDatabase(dbFile string) (retDB *sql.DB, retErr error) {
    db, err := sql.Open("sqlite", dbFile)
    if err != nil { return nil, fmt.Errorf("open: %w", err) }
    defer func() {
        if retErr != nil { db.Close() }
    }()
    // ... remaining setup; any return error triggers defer close
    return db, nil
}
```

---

#### G-2 — HIGH: `AssignGroupStationScore` — TOCTOU SELECT + INSERT/UPDATE
**File:** `backend/database/scores.go`

The current SELECT → INSERT/UPDATE is non-atomic (two round trips, time-of-check-to-time-of-use race). Replace with a single upsert that the existing `UNIQUE(group_id, station_id)` constraint supports:

```go
_, err := db.Exec(`
    INSERT INTO group_station_scores (group_id, station_id, score)
    VALUES (?, ?, ?)
    ON CONFLICT(group_id, station_id) DO UPDATE SET score = excluded.score`,
    groupID, stationID, score)
return err
```

---

#### G-3 — HIGH: `map[string]interface{}` as Universal Return Type
**Files:** `app_handlers.go`, all `backend/handlers/*.go`

Every handler returns `map[string]interface{}`. Key names are stringly-typed (`"status"`, `"message"`, `"count"`): a typo is invisible to the compiler and to IDEs. A typed response envelope would restore type safety:

```go
type Response struct {
    Status  string `json:"status"`
    Message string `json:"message,omitempty"`
}
type DataResponse[T any] struct {
    Response
    Data T `json:"data,omitempty"`
}
```

#### G-4 — MEDIUM: Package Named `io` Shadows stdlib
**Directory:** `backend/io/`

The package name `io` shadows the Go standard library `"io"` package. Two handler files already work around this with import aliases (`iolib "io"`). Rename to `report`, `pdfgen`, or `export`.

#### G-5 — MEDIUM: `context.Context` Stored in Struct Field
**File:** `main.go`

```go
type App struct {

```

The Go documentation explicitly states: *"Do not store Contexts inside a struct type."* `a.ctx` is only used in `LoadFile` for `runtime.OpenFileDialog`. Pass it as a method parameter.


This hidden global mutable state creates an invisible call-order dependency. Pass the output directory as a parameter to all PDF generation functions.

#### G-7 — MEDIUM: Database Errors Returned Without `fmt.Errorf` Wrapping
**Files:** `backend/database/evaluations.go`, `queries.go`, `scores.go`

Bare `return nil, err` from `rows.Scan` and `Exec` lose all context. Log-search for the source of a DB error becomes guesswork. Use `fmt.Errorf("scanning group evaluation: %w", err)` at every error site.

#### G-8 — MEDIUM: `UseExistingDB` Discards `COUNT(*)` Error
**File:** `backend/handlers/files.go`, line ~41

```go

_ = (*db).QueryRow("SELECT COUNT(*) FROM teilnehmende").Scan(&count)
return map[string]interface{}{"status": "ok", "count": count}
```

A DB error returns `count: 0`, which the frontend interprets as "empty database" rather than "error". Handle the error explicitly.

#### G-9 — MEDIUM: `HasScores` Discards DB Error
**File:** `backend/handlers/files.go`, line ~253

```go
func HasScores(db *sql.DB) bool {
    var count int
    _ = db.QueryRow(...).Scan(&count)
    return count > 0  // returns false on error
}
```

A DB failure produces `false`, which can allow the UI to enable the "Gruppen zusammenstellen" button in a state where scores already exist. Change to `(bool, error)` and handle the error in callers.

#### G-10 — MEDIUM: `a.db` Replaced Under Potential Concurrent Goroutines
**File:** `app_handlers.go`

Four handlers accept `**sql.DB` and replace the underlying pointer (`UseExistingDB`, `ResetToFreshDB`, `LoadFile`, `RestoreDatabase`). Wails can dispatch concurrent IPC calls; without a `sync.RWMutex` protecting `a.db`, concurrent replacement is a data race. Add a mutex to the `App` struct.

#### G-11 — MEDIUM: `startup()` Continues Silently After Config Load Failure
**File:** `main.go`, lines ~39–42

```go
cfg, err := config.LoadOrCreate()
if err != nil {
    fmt.Printf("Konfiguration konnte nicht geladen werden: %v\n", err)
```

There is no user notification. Consider `runtime.MessageDialog` for critical startup errors.

#### G-12 — MEDIUM: No `context.Context` in Any Database Operation
**Files:** All `backend/database/*.go`

All DB calls use non-context variants (`db.Query`, `db.Exec`, `db.QueryRow`). Go 1.8+ provides context-aware equivalents. Without them, Wails shutdown cannot cancel in-flight queries. Use `db.ExecContext(ctx, ...)` throughout.

## Security Issues (All Layers)

### SEC-1 — HIGH: XSS via Unescaped `dbName` in `innerHTML`
**File:** `frontend/app.js`, `_handleStartupDBChoice()`, line ~45

```js
box.innerHTML = `... <strong>${dbName}</strong> ...`;
```

`dbName` is the raw database filename from the Go backend. A filename like `<img src=x onerror=alert(1)>.db` would execute arbitrary JavaScript in the Wails WebView2 context. (Windows NTFS file names may contain `<` and `>`.)

**Fix:** Write the static HTML shell first, then:
```js
box.querySelector('#_dbNameSpan').textContent = dbName; // XSS-safe
```

---

### SEC-2 — HIGH: XSS via Unescaped Backup Filename in Restore Dialog
**File:** `frontend/admin/file-handler.js`, lines 109–111

```js
dialogHTML += '<div ...>' + backup.name + '</div>';
dialogHTML += '<span>📅 ' + backup.modified + '</span>';
```

`backup.name` is a filesystem path. A file named `<script>alert(1)</script>.db` is injectable. Apply `escapeHtml()` consistently with the rest of the codebase.

---

### SEC-3 — HIGH: JavaScript Injection via `backup.name` in `onclick` Attribute
**File:** `frontend/admin/file-handler.js`, line 108

```js
dialogHTML += 'onclick="window.selectBackup(' + index + ', \'' + backup.name + '\')">';
```

A filename containing a single quote (legal on Windows) terminates the JS string literal and allows arbitrary JS insertion. Payload example: `'); window.go.main.App.ResetToFreshDB(); ('`.

**Fix:** Use `data-*` attributes and `addEventListener`:
```js
itemEl.dataset.backupName = backup.name;  // .dataset auto-escapes
itemEl.addEventListener('click', () => window.selectBackup(index, backup.name));
```

---

### SEC-4 — HIGH: Path Traversal in `RestoreDatabase`
**File:** `backend/handlers/backup.go`, line ~68

(Detailed under A-2 above.)

```go
backupPath := filepath.Join("dbbackups", backupFilename)
// "../config.toml" becomes "config.toml"
```

Apply the same `filepath.Abs` + `strings.HasPrefix` containment check already used in `GetGroupPictureAsBase64`.

---

### SEC-5 — MEDIUM: Unescaped Numeric Field `t.Alter` in `groups.js`
**File:** `frontend/groups/groups.js`, line ~79

```js
html += '<td>' + t.Alter + '</td>';  // no escapeHtml
```

All adjacent cells use `escapeHtml`; this one is inconsistent. If the backend ever returns a non-numeric string (Excel edge case), it is injected raw. Fix: `escapeHtml(String(t.Alter ?? ''))`.

## Accessibility Review

### ACC-1 — HIGH: Missing `lang` Attribute on `<html>`
**File:** `frontend/index.html`, line 2

```html
<html>  <!-- missing lang -->
```

Screen readers use `lang` to select speech synthesis language/engine. Fix:
```html
<html lang="de">
```

---

### ACC-2 — HIGH: No `aria-live` on Status Bar
**File:** `frontend/index.html`, status div

`setStatus()` updates the status bar constantly but there is no `aria-live` region. Screen readers are not notified of status changes (progress, errors, success).

```html
<div class="status" id="status" role="status" aria-live="polite">Bereit...</div>
```

Use `aria-live="assertive"` for error messages.

---

### ACC-3 — HIGH: Modal Dialogs Lack ARIA Dialog Semantics and Focus Trap
**Files:** `frontend/app.js` (startup dialog), `frontend/admin/file-handler.js` (restore dialog), `frontend/admin/config-editor.js`

None of the three modal implementations have:
- `role="dialog"` and `aria-modal="true"`
- `aria-labelledby` referencing the modal heading
- Focus moved into the modal on open
- Focus trap (Tab stays inside the modal)
- Escape key to close

Minimum fix per modal:
```js
overlay.setAttribute('role', 'dialog');
overlay.setAttribute('aria-modal', 'true');
overlay.setAttribute('aria-labelledby', 'modal-heading-id');
overlay.querySelector('button, input, select').focus();
overlay.addEventListener('keydown', e => { if (e.key === 'Escape') closeModal(); });
```

### ACC-7 — LOW: Decorative Emoji Not Hidden from Screen Readers
The `<h1>`, buttons, and generated HTML throughout the app use emoji (🎯, 📝, 📊 etc.) as decorations. Screen readers announce them verbatim. Wrap decorative emoji in `<span aria-hidden="true">`.

### ACC-8 — LOW: `confirm()` / `alert()` Calls
`confirm()` is used in `file-handler.js` and `stations.js`. These are not accessible, cannot be styled, and look inconsistent in WebView2. Replace with the custom modal pattern already demonstrated in the restore dialog.

`frontend/index.html` has no `<main>` element or `role="main"` landmark. Screen reader users cannot jump to the main content area.
